O que é o Hexo?

O Hexo é uma ferramenta simples, rápida e poderosa para criação blog.

Por que usar?

  • Feito em node.js
  • Documentação completa em vários idiomas
  • Diversos plugins e temas
  • Deploy simples para Githubpages, Heroku, etc.

Instalação do hexo

Hexo

npm install -g hexo-cli

hexo-cli

O hexo-cli é um pacote que nos permite criar o nosso projeto, posts, rascunhos e páginas por linha de comando.

Estrutura do Projeto

Arquivo Descrição
node_modules/ Todas as dependências necessárias para o projeto.
scaffolds/ Templates das páginas de conteúdo.
source/ Onde ficam os posts, rascunhos e páginas.
themes/ Onde ficam os temas do projeto.
_config.yml Arquivo de configuração do projeto.
.gitignore Lista de arquivos e pastas que não irão para o github.
package.json Arquivo de descrição do projeto. Possui informações como o nome, autor, versão e as dependências.

Comandos

Comando O que faz?
hexo init nome-do-projeto Criar o projeto
hexo new nome-do-post Cria um post para o blog
hexo new draft nome-do-draft-post Cria um post de rascunho para o blog
hexo publish nome-do-draft-post Publica o post de rascunho para o blog
hexo new page nome-da-pagina Cria uma nova página
hexo server Subir um servidor local
hexo server --draft Subir um servidor local junto com os rascunhos

Front Matter

Front-matter é um bloco de YAML ou JSON no início do arquivo que é usado para definir configurações para o conteúdo que será escrito (como páginas ou postagens). O Front-matter é terminado por três traços quando escrito em YAML ou três ponto e vírgula quando escrito em JSON.

Scaffolds

São templates que as páginas de conteúdo irão usar, com informações vindo por padrão.

Tags e Categorias

Usadas para agrupar conteúdo em seu projeto Hexo.

Criar tags


              tags:
              - tag1
              - tag2

Criar categorias


              categories:
              - cat1
              - cat2

Criar subcategorias


              categories: [cat1, cat1.1]

Tag Plugins

Trechos de código em que você pode adicionar ao seu arquivo markdown que permite adicionar um código específico sem ter que adicionar um código complexo.

Trecho de código


                {% codeblock lang:javascript %}
                  console.log('Hello World!');
                {% endcodeblock %}

Vídeo do youtube


              {% youtube id-do-video %}

Assets folder

Pastas para colocar conteúdos que podemos usar em nosso markdown.

Criando uma asset folder


              mkdir source/assets

Importar uma imagem com Markdown


              ![legenda da imagem](/assets/imagem.jpg)

Temas

https://hexo.io/themes/

Plugins

https://hexo.io/plugins/

Instalar o plugin Hexo-admin


                npm install --save hexo-admin

Deploy para o Github Pages

Instale o pacote hexo-deployer-git


                npm install hexo-deployer-git --save

Configurar o _config.yml


              deploy:
                type: git
                repo: https://github.com/seu-usuario/seu-usuario.github.io

Colocar nosso projeto em outra branch


              git checkout -b source

Adicionar todo o projeto em stage


              git add .

Criar um commit


              git commit -m "Projeto inicial"

Fazer o push para a branch source


              git push origin source

Deploy em um comando


                hexo deploy
